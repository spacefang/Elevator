# 项目目录结构约定（建议）

目标：让 Spring Boot 后端与 Vue 前端可以并行开发，且本地依赖可一键启动。

## 推荐最终结构（monorepo）

```
.
├── backend/                 # Spring Boot（后端）
├── frontend/                # Vue（前端）
├── deploy/                  # docker-compose、环境变量示例
└── docs/                    # 业务/架构/接口等文档
    └── design/              # 原始设计文档（模块/流程/权限/存储等）
```

## 与 git worktree 的配合（你现在的用法）

`git worktree` 只是一种本地并行开发手段（不建议提交到仓库）。如果你需要同时开多个分支做开发/联调，可以在本机维护一个 `worktrees/` 目录，例如：

```
worktrees/
├── backend/   # backend 分支工作区
└── frontend/  # frontend 分支工作区
```

说明：
- 当前仓库推荐以 monorepo 为主（`backend/`、`frontend/`、`docs/` 同时存在于同一分支）。
- `worktrees/` 目录仅用于本地开发便利，可忽略不提交（见根目录 `.gitignore`）。

# 项目目录结构约定（建议）

目标：让 Spring Boot 后端与 Vue 前端可以并行开发，且本地依赖可一键启动。

## 推荐最终结构（monorepo）

```
.
├── backend/                 # Spring Boot（后端）
├── frontend/                # Vue（前端）
├── deploy/                  # docker-compose、环境变量示例
└── docs/                    # 业务/架构/接口等文档
    └── design/              # 原始设计文档（模块/流程/权限/存储等）
```

## 与 git worktree 的配合（你现在的用法）

你已经有两个 worktree 目录用于并行开发（本地目录，不建议提交到仓库）：

```
worktrees/
├── backend/   # backend 分支工作区
└── frontend/  # frontend 分支工作区
```

约定：
- `deploy/`、`docs/`：尽量保持一致（需要时从 `master` 合并到 `backend/frontend` 分支）。
- `backend/`：只在 `backend` 分支上迭代后端代码。
- `frontend/`：只在 `frontend` 分支上迭代前端代码。

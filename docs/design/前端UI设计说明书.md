# 前端 UI 设计说明书

## 1. 设计概述

本设计旨在打造一个专业、现代、高效的智能电梯运维平台。界面风格偏向 B 端管理系统，强调信息展示的清晰度和操作的便捷性。

## 2. 全局风格定义

### 2.1 配色方案
- **主色调 (Primary)**: `#409EFF` (科技蓝) - 用于强调、选中、行动按钮。
- **背景色 (Background)**:
  - 页面背景: `#F5F7FA` (浅灰)
  - 内容背景: `#FFFFFF` (纯白)
- **文字颜色 (Text)**:
  - 主要文字: `#303133`
  - 常规文字: `#606266`
  - 次要文字: `#909399`
- **功能色 (Status)**:
  - 成功 (Success): `#67C23A` (绿)
  - 警告 (Warning): `#E6A23C` (橙) - 对应“橙色告警”
  - 危险 (Danger): `#F56C6C` (红) - 对应“红色告警”
  - 信息 (Info): `#909399` (灰)

### 2.2 布局结构 (Layout)
采用经典的“侧边栏 + 顶部导航”布局。
- **Sidebar (左侧)**: 
  - Logo 区 (顶部)
  - 多级导航菜单 (运维概览、告警管理、设备管理等)
  - 收缩/展开按钮
- **Header (顶部)**:
  - 面包屑导航 (Breadcrumb)
  - 全局搜索
  - 消息通知 (告警铃铛)
  - 用户头像 & 下拉菜单 (个人中心、退出登录)
- **Main Content (主体)**:
  - 标签页导航 (Tags View) - 可选
  - 内容区域 (Padding 20px, 卡片式设计)

## 3. 核心页面详细设计 (MVP)

### 3.1 登录页 (Login)
- **背景**: 或者是简约的渐变色，或者是带有科技感的电梯/城市背景图。
- **登录框**: 居中显示。
  - 标题: "智能电梯运维平台"
  - 输入项: 用户名、密码、验证码 (可选)。
  - 按钮: "登录" (全宽)。
  - 底部: 版权信息。

### 3.2 告警列表页 (Alarm List) 
**对应 `docs/协作进度.md` - P0**

- **筛选区 (Filter)**:
  - 顶部一行搜索栏。
  - 输入框: 设备编号/位置。
  - 下拉框: 告警级别 (红/橙/黄)、状态 (待处理/处理中/已完成)。
  - 按钮: "查询"、"重置"。
- **数据表格 (Table)**:
  - 字段: 级别 (带颜色Tag)、类型、设备编号、位置、内容、触发时间、持续时长、状态、负责人。
  - 操作列: "详情"、"处理/派单"。
- **分页 (Pagination)**: 底部右对齐。

### 3.3 告警详情页 (Alarm Detail)
**对应 `docs/协作进度.md` - P0**

- **头部信息**:
  - 左侧: 告警标题 (如 "困人告警 - 设备 #1001")。
  - 右侧: 告警级别 Tag、当前状态 Tag、触发时间。
- **布局分栏**:
  - **左侧 (60%) - 告警与设备信息**:
    - **基本信息卡片**: 告警内容、阈值、可能原因。
    - **设备档案卡片**: 品牌、型号、位置、维保负责人。
    - **实时监控卡片**: (占位) 预留视频流或当前楼层/状态显示。
  - **右侧 (40%) - 处理流程**:
    - **处理记录时间轴**: 显示从触发到当前的每一步操作 (自动/人工)。
    - **操作面板**: 
      - 按钮: "接单" (如未接)、"完成处理"、"转工单"。
      - 备注输入框。

## 4. 交互体验
- **加载状态**: 表格/详情数据加载时显示 Skeleton 或 Loading Spinner。
- **反馈提示**: 操作成功/失败均有 Message Toast 提示。
- **导航高亮**: 当前路由对应的菜单项高亮显示。
